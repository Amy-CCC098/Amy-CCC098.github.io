kali-linux系统文件监控【python】

系统监控的意义

随着大数据、各种电脑技术的发展，信息的安全性成了一个很重要的课题。文件监控系统可以帮助我们监视某个目录下文件的变化，包括增加、删除、修改，从而使本地用户可以在指定目录下的文件夹内容发生变动时及时知道。

文件监控系统可以在安全攻防中进行简单的防护，防止恶意攻击者对文件的新建、删除和修改。同时也可以在日常生活中进行监控，保障数据文件的安全，有了它用户就可以完全放心在电脑中存储文件，不用担心会有人动用或者是改变文件。数据文件是非常宝贵的资产，有些时候，如果数据发生了变化,而没有被意识到,这是非常危险的，比如医疗系统等。

可行性研究

文件监控系统在保障数据文件等方面具有重要意义。目前，有了较为成熟的实现方法。

功能需求分析

（1）程序基于kali linux系统，需要实现对某一目录下的文件变动进行实时监控，并且将相应的改动以文字的方式呈现出来，比如目录下增加了文件:xx.doc，修改了文件:xx.doc等。具体监控内容可以分为监控新增文件、监控删除文件、监控文件修改。

（2）将每一次的改动记录存储到一个表格中，以便之后的检查。

（3）可以监控一段时间的文件变动

非功能需求分析

（1）性能要求：响应速度要做到尽量快，以保证实时监控

（2）可靠性安全性：文件内容用md5加密

（3）运行要求：基于kali linux系统

（4）界面要求：可视化或者将监控数据存入表格

方案设计

实现原理

我们可以通过监控系统内文件信息长度的先后比较来判断是否新增或删除，通过比较相同路径文件的key对应的值是否相同来判断文件是否有修改。

（1）初始化

首先是初始化各类参数，包括需要需要监控的文件路径，字体颜色等。还设置两个字典file_info和found_info来存储目录下所找到的文件路径。

（2）搜索文件

设置一个file_finder()函数用于寻找路径中的所有文件，返回md5加密的文件路径。并将找到的所有文件路径赋给found_info，如果没有改动的操作，found_info再将文件路径赋值给file_info。

（3）检测是否有文件新增行为

当file_finder()再次找到的found_info的长度大于先前的file_info时，说明新增文件。新增的文件路径可以通过file_info和found_info异或来获得。最后print

（4）检测是否有文件删除行为

当file_finder()再次找到的found_info的长度小于先前的file_info时，说明删除了某个文件。删除的文件路径也可以通过file_info和found_info异或来获得。最后print

（5）检测是否有文件修改行为

由于file_info和found_info里的数据都以字典的形式存储，可以通过判断两者相同的key对应的值是否相同来判断是否有修改。如果相同，未被修改。如果不同，则被修改，print

（6）将监控到的内容导入一张表格中

最后通过csv.writer将所有监控的数据导出到一个csv表格文件中，方便之后的观察。
